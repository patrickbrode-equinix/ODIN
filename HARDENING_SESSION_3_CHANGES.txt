HINWEISE FÜR ORACLE LINUX VM + PORTAINER DEPLOYMENT
============================================================

1. Log-Driver:
   Container nutzen journald → "podman logs" liefert leere Ausgabe.
   Für Portainer/Oracle: log-driver auf json-file oder k8s-file setzen:
   In podman-compose.wsl.yml unter jedem Service ergänzen:
     logging:
       driver: json-file

2. DB-Volume:
   merged_postgres_data muss persistent bleiben.
   NIEMALS "podman system prune -v" auf Produktion ausführen.

3. Migrations (Frische Installation):
   Alle 12 Migrations nutzen CREATE TABLE IF NOT EXISTS → vollständig idempotent.
   Auf frischer DB laufen alle 12 automatisch durch.

4. DB_PASS Warning:
   Startup zeigt "DB_PASS: ⚠ MISSING" — dies ist normal wenn DATABASE_URL
   gesetzt ist. In Portainer als Secret/Env-Var setzen:
     DATABASE_URL=postgresql://postgres:<passwort>@odin-postgres:5432/odin

5. JWT_SECRET:
   Aktuell "dev" (unsicher). Für Produktion zwingend ändern:
     JWT_SECRET=<zufälliger 256-bit Wert>

6. Postgres depends_on:
   Backend hat bereits Retry/Backoff (10 Versuche × 5s = 50s max).
   Zusätzlich in compose: depends_on: odin-postgres: condition: service_healthy
